# Orthanc MWL Integration (MediOffice)

This project now supports radiology requests that generate MWL entries for Orthanc.

## 1) Required MediOffice env

Set these values in `.env`:

- `ORTHANC_BASE_URL=http://127.0.0.1:8042`
- `ORTHANC_USERNAME=orthanc`
- `ORTHANC_PASSWORD=orthanc`
- `ORTHANC_WORKLIST_PATH=/worklists`
- `ORTHANC_WORKLIST_DIRECTORY=storage:app/orthanc/worklists`
- `ORTHANC_DUMP2DCM_PATH=C:\dcmtk\bin\dump2dcm.exe` (optionnel, recommande sous Windows)
- `ORTHANC_WEBHOOK_TOKEN=<shared-secret>`
- `DICOM_UID_ROOT=1.2.826.0.1.3680043.10.5432`

## 2) Orthanc plugin setup

Enable the Modality Worklists plugin and point it to the same folder used by `ORTHANC_WORKLIST_DIRECTORY`.

Example (Orthanc `Configuration.json`):

```json
{
  "Plugins": [
    "/usr/share/orthanc/plugins"
  ],
  "Worklists": {
    "Enable": true,
    "Database": "/var/lib/orthanc/worklists"
  }
}
```

On Windows, use a local folder path and make sure Orthanc service account can read it.

## 3) DICOM tags generated

The worklist dataset includes at minimum:

- `(0010,0010)` PatientName
- `(0010,0020)` PatientID (MRN)
- `(0020,000D)` StudyInstanceUID (generated by MediOffice)
- `(0040,0100)` ScheduledProcedureStepSequence
- `(0008,0050)` AccessionNumber (unique reconciliation key)
- `(0032,1060)` RequestedProcedureDescription
- `(0040,0001)` ScheduledStationAETitle

## 4) Workflow status mapping

`RadiologyRequest.workflow_status` values:

- `requested` = Demandee (in worklist)
- `in_progress` = Manipulateur a pris en charge
- `received` = Images recues par Orthanc
- `completed` = Examen termine

## 5) Orthanc -> MediOffice webhook

Endpoint:

- `POST /care/module-3/orthanc/webhook`
- Header: `X-Orthanc-Token: <ORTHANC_WEBHOOK_TOKEN>`

Body example:

```json
{
  "event": "study-received",
  "accession_number": "ACC20260420000123",
  "orthanc_study_id": "3f8f9dc2-3bb9b6a8-..."
}
```

Supported events:

- `study-started` -> `in_progress`
- `study-received` -> `received`
- `study-completed` -> `completed`

## 6) Notes on .wl generation

MediOffice always writes a worklist payload file in the configured directory.

- Default output is `*.wl.json`.
- If `dump2dcm` (dcmtk) is available on PATH, MediOffice also tries to generate a native `*.wl` file.

This keeps integration operational even when dcmtk is not installed.
